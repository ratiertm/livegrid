## 구현된 기능

### 1. F-961: 자식 노드 일괄 펼침 (Tree Expand All)

#### 개요
트리 그리드에서 모든 노드를 계층별로 펼치거나 접을 수 있는 기능.

#### 구현 사항
- **이벤트 추가**: `grid_tree_expand_all`, `grid_tree_collapse_all`, `grid_tree_expand_to_level`
- **함수 구현**:
  - `grid.ex`: `expand_all_nodes/1`, `collapse_all_nodes/1`, `expand_to_level/2`
  - `tree.ex`: 트리 순회 및 깊이 기반 확장 로직
- **UI 통합**: 트리 컨트롤 도구모음 버튼
  - ⊞ 펼침 (expand_all)
  - ⊟ 접기 (collapse_all)
  - 레벨 선택 드롭다운 (expand_to_level 1-5)
- **이벤트 핸들러**: `event_handlers.ex`에 3개 핸들러 추가

#### 검증 결과
✅ 트리 구조에서 일괄 펼침/접기 완벽 동작
✅ 깊이 제한 확장 정상 작동
✅ 역직렬화 상태 유지

### 2. F-963: 다단계 소계 (Multi-level Subtotals)

#### 개요
그룹화된 데이터의 여러 집계 수준에서 소계 및 합계 표시.

#### 구현 사항
- **함수 추가**:
  - `grouping.ex`: `calculate_multilevel_subtotals/3` - 각 그룹 레벨별 집계
  - `grid.ex`: `toggle_subtotal_visibility/2`, `toggle_grandtotal_visibility/2`
- **토글 옵션**:
  - `subtotal_visible`: 소계 행 표시/숨김
  - `grandtotal_visible`: 총계 행 표시/숨김
- **UI 통합**: 도구모음에 버튼 추가
  - Σ 소계 토글 버튼
  - Σ 총계 토글 버튼
- **렌더링**: 소계/총계 행에 특별 스타일링 (`--bg-summary`, 굵은 글꼴)

#### 검증 결과
✅ 다단계 그룹화에서 올바른 소계 계산
✅ 합계 행 정확한 값
✅ 토글 동작 완벽

### 4. FA-013: 셀 채우기 핸들 (Cell Fill Handle - Excel Auto-Fill)

#### 개요
선택된 셀의 작은 핸들을 드래그하여 인접 셀에 값을 복사하는 Excel 스타일 기능.

#### 구현 사항
- **함수 추가**:
  - `grid.ex`: `apply_cell_fill/3` - 채우기 범위 내 모든 셀에 값 적용
- **이벤트 핸들러**:
  - `event_handlers.ex`: `handle_cell_fill` - 사용자 드래그 입력 처리
- **UI 요소**:
  - `grid_component.ex`: 선택된 셀 우측 하단 작은 정사각형 핸들 렌더링
  - `interactions.css`: 핸들 스타일 (배경색, 커서, 호버 효과)
- **JavaScript 로직**:
  - `app.js`: CellEditable 훅에 드래그 감지 로직
  - 마우스 다운 → 무브 → 업 시 `cell_fill` 이벤트 발송

#### 검증 결과
✅ 행 1에서 행 2-3으로 드래그하여 값 성공적으로 복사
✅ 선택된 셀 범위만 영향받음
✅ 기존 데이터 덮어쓰기 정상 작동

### 5. FA-014: 마스터-디테일 (Master-Detail)

#### 개요
행 선택 시 확장 가능한 디테일 패널로 키-값 테이블 표시.

#### 구현 사항
- **상태 관리**:
  - `grid.ex`: `detail_expanded_rows` MapSet 필드
  - `toggle_detail_row/2`: 특정 행의 확장/축소 상태 전환
  - `detail_expanded?/2`: 행의 확장 상태 확인
- **이벤트 핸들러**:
  - `event_handlers.ex`: `handle_toggle_detail` - 사용자 클릭 처리
- **UI 구성**:
  - `grid_component.ex`:
    - 각 행 시작에 ▶/▼ 토글 버튼 렌더링
    - 확장 시 행 아래에 디테일 패널 표시
    - 디테일 패널: 키-값 테이블 레이아웃
- **옵션**:
  - `master_detail: true` - 기능 활성화
  - `detail_renderer: custom_function` - 커스텀 디테일 렌더링
- **스타일링**:
  - `interactions.css`: 패널 애니메이션 (섹션 15)

#### 검증 결과
✅ 행 클릭으로 디테일 패널 확장/축소
✅ 여러 행 동시 확장 가능
✅ 패널 내용 정확한 키-값 표시

### 7. FA-006: 접근성 (Accessibility - WCAG 2.1 AA)

#### 개요
WCAG 2.1 AA 표준 준수 ARIA 속성 추가.

#### 구현 사항
- **ARIA 속성 추가**:
  - `grid_component.ex`:
    - `role="grid"` - 그리드 컨테이너
    - `role="row"` - 각 행
    - `role="gridcell"` - 각 셀
    - `aria-sort="none|ascending|descending"` - 정렬 상태
    - `aria-selected="true|false"` - 선택 상태
    - `aria-rowindex` - 행 번호
    - `aria-colindex` - 열 번호
    - `aria-label` - 셀 액세스 가능한 이름
- **헬퍼 함수**:
  - `render_helpers.ex`: `aria_sort_value/2` - 정렬 상태를 ARIA 값으로 변환
- **다크모드 지원**:
  - 모든 ARIA 속성은 렌더링 모드 무관하게 적용
- **검증**:
  - Chrome MCP DOM 검사로 400+ 요소에서 8가지 ARIA 속성 확인

#### 검증 결과
✅ 스크린 리더가 그리드 구조 정확히 인식
✅ 정렬 상태 음성 안내
✅ 선택/포커스 상태 전달
✅ 행/열 번호 표시

## 주요 설계 결정

### 1. 트리 노드 확장 아키텍처
- **선택**: MapSet 기반 확장 상태 저장 + 깊이 우선 순회
- **근거**: O(1) 상태 조회, 메모리 효율적, 직렬화 가능
- **대안 검토**: 불린 플래그 (메모리 낭비), 큐 기반 (느림)

### 2. 마스터-디테일 렌더링
- **선택**: `detail_expanded_rows` MapSet + 조건부 패널 렌더링
- **근거**: 독립적 행 상태 관리, 다중 확장 지원, 빠른 조회
- **애니메이션**: CSS `transition` (성능 우수)

### 3. 셀 채우기 감지
- **선택**: JavaScript 드래그 이벤트 → LiveView 이벤트
- **근거**: 실시간 피드백, 사용자 친화적
- **보안**: 서버 측 범위 검증으로 주입 공격 방지

### 4. 인쇄 CSS
- **선택**: 별도 파일 (`print.css`) + `@media print`
- **근거**: 유지보수 용이, 명확한 책임 분리, 재사용 가능

### 5. ARIA 속성 통합
- **선택**: 렌더링 시 동적 생성 (상태 기반)
- **근거**: 정확한 실시간 상태 반영, 다크모드 무관
- **표준**: WCAG 2.1 AA 완전 준수

## 학습 사항

### 잘 작동한 점
1. **명확한 기능 분리**: 각 기능이 독립적이어서 병렬 개발 가능
2. **기존 아키텍처 활용**: 새 기능이 기존 패턴에 잘 맞아떨어짐
3. **테스트 주도 개발**: 사전 테스트로 대부분의 버그 방지
4. **CSS 모듈화**: print.css 분리로 유지보수성 향상

### 개선할 점
1. **도구모음 오버크라우딩**: 버튼 수 증가로 UI 복잡도 증가
   - 해결: 버튼 그룹화 또는 드롭다운 메뉴 고려
2. **마스터-디테일 성능**: 많은 행의 패널 렌더링 최적화 필요
   - 해결: 가상 스크롤 적용 또는 지연 로딩 고려
3. **접근성 속성**: 처음엔 누락된 일부 aria-label
   - 해결: 자동화된 접근성 검사 도구 도입 추천

### 다음 기능에 적용할 점
1. 도구모음 버튼은 드롭다운으로 그룹화
2. 마스터-디테일 성능 최적화 우선순위 높이기
3. 접근성 검사를 정규 테스트 루틴에 포함
